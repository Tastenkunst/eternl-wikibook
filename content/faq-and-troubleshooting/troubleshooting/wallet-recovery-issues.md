# Wallet Recovery Issues

If you are trying to restore your wallet and encounter problems, such as error messages or a zero balance, please check these common causes:

### Verification of the Recovery Phrase
* **Problem:** The wallet cannot be restored, or Eternl says a word is invalid.
* **Solution:** 
    * Ensure you have selected the correct number of words (usually 24 for modern Shelley wallets).
    * Check for typos. All words must be part of the BIP-39 standard list. If a word is highlighted in red, it is not a valid recovery word.
    * Make sure you entered the words in the exact order they were written down.

### Zero Balance / Missing Funds
* **Problem:** The wallet restores successfully, but the balance is 0 ADA.
* **Cause - Wrong Account Index:** In Cardano, one recovery phrase can manage multiple accounts. If you previously added a "Manual Account" with a specific index (e.g., Account #1 instead of #0), you must manually add that account again in Eternl to see your funds.
* **Cause - Wrong Phrase:** It is common to have multiple recovery phrases from different wallets. Double-check if the phrase you are using is actually the one belonging to the wallet with the funds.

### Hardware Wallet Restoration
* **Problem:** You are trying to restore a phrase that was generated by a Hardware Wallet (Ledger/Trezor).
* **Solution:** 
    * Hardware wallets often use different derivation paths (algorithms) than software wallets. To access your funds securely, use the **"Connect Hardware Wallet"** option instead of "Restore Wallet."
::: warning
**Crucial Security Note:** You should **never** type your Hardware Wallet recovery phrase into a computer or any software wallet. This defeats the purpose of the hardware device's security.
:::

### Missing Funds from other Wallets (Passphrase)
* **Problem:** You are migrating from a wallet like AdaLite where you used an optional "Passphrase", but your balance in Eternl is zero.
* **Explanation:** Some third-party wallets allowed an optional passphrase (an extra word) for software wallets. However, this is not part of the official Cardano (CIP) standards for software wallets.
* **Eternl's Position:** To ensure maximum compatibility and security across the Cardano ecosystem, Eternl adheres strictly to official Cardano standards. Therefore, it does not support this non-standard passphrase layer for software-based restores.
* **Solution:** If your funds are tied to such a passphrase, you may need to use the original application to access them or move them to a standard Cardano wallet.

### Security Breach (Stolen Funds)
* **Problem:** The wallet is restored, but the transaction history shows that your funds were sent to an unknown address without your consent.
* **Cause:** If you ever stored your recovery phrase digitally (photo, cloud, email) or gave it to someone (e.g., a fake support site), your wallet is compromised.
* **Action:** If there are any remaining funds, send them to a new, secure wallet immediately. A compromised recovery phrase can never be used safely again.
